# 手把手：在本机 Windows 上第一次跑通

这篇文档只做一件事：**按顺序复制命令**，让你在**当前这台电脑**上装好 TulipBridge，并在手机里能导入连接。

如果你已经会命令行，可以直接看 [README.md](../README.md)；这里写得更细。

**人在国内的家属不能和你实时联调？** 你可先在荷兰/本机按本文做完，再把订阅文件发给对方；完整分工见 [PREP_AND_CHINA_HANDOFF_ZH.md](PREP_AND_CHINA_HANDOFF_ZH.md)。

### 进度核对（第一次跑通）

请你**从上到下**，做完一项在心里打个勾：

| 序号 | 内容 | 在哪里 |
|------|------|--------|
| □ | Python 版本 ≥ 3.10 | 下文 **第一步** |
| □ | 已 `cd` 到仓库目录 | 下文 **第二步** |
| □ | 已执行 `pip install -e .` | 下文 **第三步** |
| □ | 已用 `ipconfig` 记下 **IPv4**（局域网 IP） | 下文 **第四步** |
| □ | 已跑完初始化，屏幕出现 **`[7/7] Done`** | 下文 **第五步**（5.1→5.4） |
| □ | 已看到 **`tulipbridge-data\subscribe\`** 里有文件 | 下文 **第六步** |
| □ | 防火墙已放行端口（或先同 WiFi 测） | 下文 **第七步** |
| □ | 手机已导入并连通 | 下文 **第八步** |

---

## 这篇教程假设什么？

- 电脑是 **Windows**。
- 你已经把本仓库放到磁盘上（例如 `c:\dev-projects\tulipbridge`）。
- 电脑能上网（第一次会从 GitHub **下载 sing-box**，需要一点时间）。

---

## 第一步：确认 Python 能用

1. 按键盘 **`Win + R`**，输入 **`cmd`**，回车，打开黑色窗口。
2. 输入下面这一行，回车：

   ```text
   python --version
   ```

3. 如果显示 **`Python 3.10`** 或更高（如 3.11、3.12），可以继续。
4. 如果提示找不到命令，请先安装 Python 3.10+（安装时勾选 **Add Python to PATH**），再重复第 2 步。

---

## 第二步：进入项目文件夹

在 **cmd** 里输入（请把路径改成你机器上真实的仓库路径）：

```text
cd /d c:\dev-projects\tulipbridge
```

以后每一条命令，都先 **`cd`** 到这里再执行。

---

## 第三步：安装本项目（只需做一次）

仍在 **cmd** 里，依次执行：

```text
python -m pip install -e .
```

等它跑完，没有大片红色报错即可。

---

## 第四步：查出本机在局域网里的 IP（给手机连家用 WiFi 时用）

手机如果连的是**和你电脑同一个家里的 WiFi**，连接地址一般用 **局域网 IP**。

1. 在 **cmd** 里输入：`ipconfig`，回车。
2. 找到 **当前正在上网** 的那块网卡（常见名字：`WLAN` 或 `以太网`）。
3. 看 **`IPv4 地址`**，形如 **`192.168.x.x`**，把这串数字**记下来**，下面叫它 **`你的局域网IP`**。

> **说明：** 下面命令里的 `--public-host` 填的就是「客户端里看到的服务器地址」。同一 WiFi 下先试局域网 IP，最简单。

---

## 第五步：初始化 sing-box（按 5.1 到 5.4 做）

下面把「一大步」拆成 **四个小步**。你只需要**真正动手敲/粘贴**的是 **5.2** 和 **5.3**。

### 5.1 先确认程序能被调用（约 10 秒）

在已 **`cd` 到仓库目录** 的前提下，输入：

```text
python -m tulipbridge --version
```

- **期望：** 打印一行带版本号的文字（例如 `tulipbridge 0.1.0`）。
- **若报错「找不到模块」：** 回到 [第三步](#第三步安装本项目只需做一次) 再执行一次 `python -m pip install -e .`。

再输入（可选，用于心里有底）：

```text
python -m tulipbridge init --help
```

- **期望：** 屏幕列出很多参数，其中能看到 **`--public-host`**、**`--port`** 等。

---

### 5.2 准备好「那一整行」命令（只改 IP）

**不要急着回车。** 先在记事本里写好下面这一行，**只做一件事**：把 **`你的局域网IP`** 换成第四步抄下来的数字（例如 `10.10.10.106`）。

```text
python -m tulipbridge --portable init --port 8443 --hy2-port 8446 --tuic-port 8447 --tls-sni tulipbridge.local --public-host 你的局域网IP --force
```

写好后检查三点：

1. **`你的局域网IP`** 已经变成纯数字和点，中间**没有空格**。
2. **整行是一条命令**，中间没有换行（若自动换行是窗口显示问题，复制时要保证是一段）。
3. 端口若你曾改过（例如不用 8443），记住后面的防火墙要和这里一致。

---

### 5.3 粘贴到 cmd 里执行，对照屏幕进度

1. 回到 **cmd**，确认当前目录仍是仓库（不确定就再执行一次 `cd /d c:\dev-projects\tulipbridge`）。
2. 粘贴 **5.2** 里那一整行，**回车**。
3. **第一次运行**可能要 **下载 sing-box**，网络慢时会多等一会儿，不要关掉窗口。

屏幕上会陆续出现 **`[1/7]` … `[7/7]`**，含义大致是：

| 显示 | 大致含义 |
|------|----------|
| `[1/7]` | 创建数据目录 |
| `[2/7]` | 准备 / 下载 sing-box |
| `[3/7]` | 生成或读取密钥 |
| `[4/7]` | 写入 `config.json` |
| `[5/7]` | sing-box 检查配置 |
| `[6/7]` | 启动 sing-box 进程 |
| `[7/7]` | 本轮初始化结束 |

若 **`[7/7] Done`** 出现后，下面还有 **「Subscription」/「订阅」/「QR」** 之类输出，说明 **`--public-host` 已生效**，订阅和二维码会写到磁盘。

---

### 5.4 怎样算成功？失败了怎么办？

**成功时至少满足：**

- 最后一屏里有 **`[7/7] Done`**。
- **没有**以 **`Error:`** 开头的一大段把流程打断。
- 若你带了 **`--public-host`**，后面还有订阅 / 二维码相关文字。

**失败时：**

- 先看 **`Error:`** 后面**第一行**写的是什么。
- **端口被占用：** 把命令里的 **`8443`** 改成例如 **`9443`**，**`8446` / `8447`** 也改成别的数字，**三条端口一起改**，再执行一次（仍可加 `--force`）。
- **sing-box 立刻退出：** 打开 **`tulipbridge-data\logs\sing-box.log`** 看最后几行；若是 **`bind`** / **`Only one usage of each socket address`**，请把本仓库 **更新到最新代码**（Windows 监听已修复），再重复 **5.2～5.3** 同一条 `init`。
- **下载失败：** 检查能否访问 GitHub、是否需要代理。

**参数说明（想弄懂再看）**

| 部分 | 通俗意思 |
|------|----------|
| `python -m tulipbridge` | 用 Python 运行 TulipBridge |
| `--portable` | 数据放在当前目录下的 **`tulipbridge-data`** |
| `--port 8443` | VLESS 用 **TCP 8443** |
| `--hy2-port 8446` | Hysteria 2 用 **UDP 8446** |
| `--tuic-port 8447` | TUIC 用 **UDP 8447** |
| `--tls-sni tulipbridge.local` | 与自签证书一致，一般不改 |
| `--public-host …` | 分享链接里的服务器地址；同 WiFi 填 **局域网 IP** |
| `--force` | 允许停掉旧 sing-box 再启动；多次尝试时建议带上 |

---

## 第六步：找到生成的文件（给手机用）

数据在仓库目录下的文件夹：

**`tulipbridge-data\subscribe\`**

里面常见有：

- **`uris-plain.txt`**：一行一条链接，可复制到 Shadowrocket。
- **`subscription.txt`**：一整段 Base64，给支持订阅导入的客户端用。
- **`qr-*.png`**：**二维码图片**。  
  **优先用这个 PNG 扫码**，比在黑色窗口里扫 ASCII 二维码更稳。

---

## 第七步：Windows 防火墙（否则手机连不上本机）

手机连你电脑上的 sing-box 时，流量是**进入本机**，要在防火墙里加 **入站规则**。端口须与 **第五步** 里一致（下面是教程默认示例；你若改过端口，这里跟着改）：

| 协议 | 端口 | 用途 |
|------|------|------|
| **TCP** | **8443** | VLESS（建议先保证这条） |
| **UDP** | **8446** | Hysteria 2 |
| **UDP** | **8447** | TUIC |

### 7.1 打开界面（与你截图一致）

任选一种：

- **`Win + R`** → 输入 **`wf.msc`** → 回车；或  
- **设置** → **隐私和安全性** → **Windows 安全中心** → **防火墙和网络保护** → **高级设置**。

左侧选中 **「入站规则」**。右侧 **「操作」** 栏里（不要选错「出站规则」那一列）点 **「新建规则…」**。

### 7.2 第一条规则：TCP 8443

1. **规则类型** → **端口** → 下一步。  
2. **协议和端口** → 选 **TCP**，**特定本地端口** 填 **`8443`** → 下一步。  
3. **操作** → **允许连接** → 下一步。  
4. **配置文件** → 至少勾选 **专用**（家里 WiFi 多为专用网络）；搞不清可先三项全勾（便于排查）→ 下一步。  
5. **名称** → 例如 **`TulipBridge VLESS TCP 8443`** → 完成。

### 7.3 第二条规则：UDP 8446 与 8447

再次点 **「新建规则…」**，重复 wizard，区别只在第 2 步：

- 选 **UDP**，**特定本地端口** 填 **`8446,8447`**（英文逗号，中间无空格）。

若系统不允许一行多个端口，就建两条：一条 UDP **`8446`**，一条 UDP **`8447`**。

名称可写 **`TulipBridge UDP Hy2 TUIC`**。

### 7.4 核对与排查

- 在入站规则列表里搜索 **`8443`** 或 **`TulipBridge`**，确认规则 **已启用**。  
- 仍不通：对照端口是否与 **init** 命令一致；必要时仅**短时间**关闭防火墙对比（**不要长期关**）。

---

## 第八步：手机上导入

1. 用微信 / 网盘 / 数据线，把 **`qr-vless-reality.png`**（或 `subscribe` 里任意一张 **PNG**）传到手机。
2. 用 **Shadowrocket**（或你文档里写的客户端）**扫描这张图片**。
3. 先测 **VLESS（TCP）** 能否上网；再必要时试 UDP 线路。

更细的客户端说明见：[SHADOWROCKET.md](SHADOWROCKET.md)。

## 中方在国内、不方便实时联调时

你这边尽量完成：**防火墙、路由器映射、`status` / `update`、生成 `subscribe/`**。对方只需在最后一步用 Shadowrocket 扫码或导入链接；具体要求与打包清单见 [PREP_AND_CHINA_HANDOFF_ZH.md](PREP_AND_CHINA_HANDOFF_ZH.md)。

## 以后只换了 IP 或域名，要不要重装？

不用。在项目目录里执行（把 **`你的新地址`** 换成新的 IP 或域名）：

```text
python -m tulipbridge --portable links --public-host 你的新地址
```

会重新生成 **`tulipbridge-data\subscribe\`** 里的链接和二维码，**不用重启 sing-box**。

---

## 和手机不在同一 WiFi（要从外面连家里宽带）时

上面第五步里，`--public-host` 要改成你家的 **公网 IP** 或 **域名**，并且在**路由器**里做 **端口映射**（外网 TCP/UDP 转到这台电脑的同样端口）。这一块每家路由器界面不同，需要你自己在路由器管理页里设置。

---

## 小词典

| 词 | 意思 |
|----|------|
| **局域网 IP** | 你家路由器分给电脑的地址，一般 `192.168.x.x`，只有同一 WiFi 里好用。 |
| **公网 IP** | 运营商给你家宽带对外的地址，别人从互联网连你需要它（或域名）。 |
| **端口** | 电脑上的「门牌号」，VLESS/Hy2/TUIC 各占不同的端口。 |
