# 手把手：在本机 Windows 上第一次跑通

这篇文档只做一件事：**按顺序复制命令**，让你在**当前这台电脑**上装好 TulipBridge，并在手机里能导入连接。

如果你已经会命令行，可以直接看 [README.md](../README.md)；这里写得更细。

---

## 这篇教程假设什么？

- 电脑是 **Windows**。
- 你已经把本仓库放到磁盘上（例如 `c:\dev-projects\tulipbridge`）。
- 电脑能上网（第一次会从 GitHub **下载 sing-box**，需要一点时间）。

---

## 第一步：确认 Python 能用

1. 按键盘 **`Win + R`**，输入 **`cmd`**，回车，打开黑色窗口。
2. 输入下面这一行，回车：

   ```text
   python --version
   ```

3. 如果显示 **`Python 3.10`** 或更高（如 3.11、3.12），可以继续。
4. 如果提示找不到命令，请先安装 Python 3.10+（安装时勾选 **Add Python to PATH**），再重复第 2 步。

---

## 第二步：进入项目文件夹

在 **cmd** 里输入（请把路径改成你机器上真实的仓库路径）：

```text
cd /d c:\dev-projects\tulipbridge
```

以后每一条命令，都先 **`cd`** 到这里再执行。

---

## 第三步：安装本项目（只需做一次）

仍在 **cmd** 里，依次执行：

```text
python -m pip install -e .
```

等它跑完，没有大片红色报错即可。

---

## 第四步：查出本机在局域网里的 IP（给手机连家用 WiFi 时用）

手机如果连的是**和你电脑同一个家里的 WiFi**，连接地址一般用 **局域网 IP**。

1. 在 **cmd** 里输入：`ipconfig`，回车。
2. 找到 **当前正在上网** 的那块网卡（常见名字：`WLAN` 或 `以太网`）。
3. 看 **`IPv4 地址`**，形如 **`192.168.x.x`**，把这串数字**记下来**，下面叫它 **`你的局域网IP`**。

> **说明：** 下面命令里的 `--public-host` 填的就是「客户端里看到的服务器地址」。同一 WiFi 下先试局域网 IP，最简单。

---

## 第五步：一次性初始化（核心命令）

把下面这一整段复制到 **cmd** 里执行。**一共只要改一个地方**：

- 把 **`你的局域网IP`** 换成你在第四步抄下来的数字（例如 `192.168.1.105`）。

```text
python -m tulipbridge --portable init --port 8443 --hy2-port 8446 --tuic-port 8447 --tls-sni tulipbridge.local --public-host 你的局域网IP --force
```

### 这一行在干什么？（可选读）

| 部分 | 通俗意思 |
|------|----------|
| `python -m tulipbridge` | 用 Python 运行 TulipBridge |
| `--portable` | 所有数据放在当前文件夹下的 **`tulipbridge-data`**，找文件方便 |
| `--port 8443` | VLESS 用 **TCP 8443**（不用 443，减少要管理员权限的情况） |
| `--hy2-port 8446` | Hysteria 2 用 **UDP 8446** |
| `--tuic-port 8447` | TUIC 用 **UDP 8447** |
| `--tls-sni tulipbridge.local` | 和自动生成的自签证书名字一致，一般不用改 |
| `--public-host …` | 生成分享链接时用的「服务器地址」，同一 WiFi 下填 **局域网 IP** |
| `--force` | 如果你以前跑过 init，这次允许它停掉旧进程重来；第一次跑也可以带上 |

### 怎样算成功？

- 屏幕里出现类似 **`[7/7] Done`**。
- 后面还有 **订阅 / 二维码** 相关输出。
- 没有停在 **`Error:`** 开头的一大段。

### 如果失败了？

- 看屏幕上 **`Error:`** 后面第一句话，常见是端口被占用：把 **`8443`** 改成 **`9443`**，**`8446/8447`** 也改成别的数字再试。
- 若 **`sing-box.log`** 里出现 **`bind`** / **`Only one usage of each socket address`**：请 **更新本仓库代码到最新**（已针对 Windows 监听地址做过修复），然后对 **同一条 `init` 命令** 再运行一次（仍需 `--force`）。
- 第一次运行若下载失败：检查网络、代理、能否打开 GitHub。

---

## 第六步：找到生成的文件（给手机用）

数据在仓库目录下的文件夹：

**`tulipbridge-data\subscribe\`**

里面常见有：

- **`uris-plain.txt`**：一行一条链接，可复制到 Shadowrocket。
- **`subscription.txt`**：一整段 Base64，给支持订阅导入的客户端用。
- **`qr-*.png`**：**二维码图片**。  
  **优先用这个 PNG 扫码**，比在黑色窗口里扫 ASCII 二维码更稳。

---

## 第七步：Windows 防火墙（否则手机连不上本机）

第一次建议手动放行端口（与第五步一致）：

- **TCP 8443**
- **UDP 8446**
- **UDP 8447**

在「Windows 安全中心 → 防火墙 → 高级设置」里添加入站规则；若暂时搞不定，可仅在测试时关闭防火墙对比是否为此原因（**不建议长期关闭**）。

---

## 第八步：手机上导入

1. 用微信 / 网盘 / 数据线，把 **`qr-vless-reality.png`**（或 `subscribe` 里任意一张 **PNG**）传到手机。
2. 用 **Shadowrocket**（或你文档里写的客户端）**扫描这张图片**。
3. 先测 **VLESS（TCP）** 能否上网；再必要时试 UDP 线路。

更细的客户端说明见：[SHADOWROCKET.md](SHADOWROCKET.md)。

---

## 以后只换了 IP 或域名，要不要重装？

不用。在项目目录里执行（把 **`你的新地址`** 换成新的 IP 或域名）：

```text
python -m tulipbridge --portable links --public-host 你的新地址
```

会重新生成 **`tulipbridge-data\subscribe\`** 里的链接和二维码，**不用重启 sing-box**。

---

## 和手机不在同一 WiFi（要从外面连家里宽带）时

上面第五步里，`--public-host` 要改成你家的 **公网 IP** 或 **域名**，并且在**路由器**里做 **端口映射**（外网 TCP/UDP 转到这台电脑的同样端口）。这一块每家路由器界面不同，需要你自己在路由器管理页里设置。

---

## 小词典

| 词 | 意思 |
|----|------|
| **局域网 IP** | 你家路由器分给电脑的地址，一般 `192.168.x.x`，只有同一 WiFi 里好用。 |
| **公网 IP** | 运营商给你家宽带对外的地址，别人从互联网连你需要它（或域名）。 |
| **端口** | 电脑上的「门牌号」，VLESS/Hy2/TUIC 各占不同的端口。 |

---

若某一步卡住，把 **从 cmd 里复制出来的完整报错**（或最后 20 行）单独发出来，便于对症排查。
